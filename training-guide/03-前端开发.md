# 前端开发：Vue 3 + TypeScript

## 从哪里改

- 新页面：`vueproject/src/views`，并在 `src/router/index.ts` 注册路由。
- 可复用区域：`src/components`；不要把同一段页面复制多份。
- 后端请求：统一放 `src/api`，页面不要到处直接调用 Axios。
- 公共请求行为：`src/utils/request.ts`。它会给非静态、非外部地址自动加 `/api`。
- 全局样式：`src/styles`；局部样式优先写在 `.vue` 文件的 `<style scoped>`。
- 类型：共用接口类型放 `src/api/types.ts` 或对应模块，少用 `any`。

## 看懂一个 Vue 文件

典型 `.vue` 文件有三段：`<script setup lang="ts">` 保存数据和逻辑，`<template>` 描述页面，`<style scoped>` 描述当前组件样式。修改时一次只处理一个目标。

推荐数据流：

```text
用户点击 → 页面函数 → src/api 中的函数 → /api 后端
                                  ↓
页面状态 ← 解析 Result/错误 ← Axios 返回
```

## 新增一个基础功能的顺序

1. 先确认后端接口方法、路径、参数和返回 JSON。
2. 在 `src/api/*.ts` 新增带类型的函数。
3. 在页面中处理 loading、成功、空数据、错误四种状态。
4. 若是新页面，添加路由和导航入口。
5. 用普通用户和管理员分别测试权限表现。
6. 执行 `pnpm build`；TypeScript 错误必须解决，不能用大量 `as any` 掩盖。

示例接口封装：

```ts
import request from '@/utils/request'

export interface ExampleItem {
  id: number
  name: string
}

export const fetchExamples = () =>
  request.get('/example/list') // 实际请求会是 /api/example/list
```

## 本项目的路由与登录

路由使用 HTML5 history，因此生产 Nginx 必须保留 `try_files $uri $uri/ /index.html`，否则刷新 `/admin/users` 会 404。登录信息和角色当前放在 `sessionStorage`；关闭标签页后会消失。

前端路由守卫只改善用户体验，不能保护数据。任何管理员功能都必须由后端再次验证权限；不能因为页面按钮被隐藏就认为接口安全。

## UI 开发约定

- 优先复用已有 `Neon*`、`AdminPagination`、`DefaultLayout` 等组件和 `src/styles` 中的主题变量。
- Element Plus 用于表单、表格、消息、分页；Tailwind 用于简单布局；避免同一属性同时被三套样式覆盖。
- 图片上传前端有 `imageCompressor.ts`，但后端仍必须校验大小、类型和文件名。
- 日期使用 Day.js，前后端以 `Asia/Shanghai` 为业务时区；测试跨日和开始/结束边界。
- 页面必须有加载中、无数据、失败提示；按钮提交期间应禁用，避免重复提交。

## 验证命令

```powershell
cd vueproject
pnpm build
pnpm preview
```

当前没有独立前端测试脚本和 lint 脚本，因此 `pnpm build` 是最低检查，不代表业务全部正确。至少手测登录、目标页面、提交/修改/删除、刷新深层路由和手机宽度。

## 排错技巧

F12 的 Console 看前端异常，Network 看请求。先记录请求 URL、方法、状态码、请求体和响应体：状态 404 多为路径错，400 多为参数错，500 看后端日志，502 多为 Nginx 找不到后端，超时可能是后端/数据库慢或当前 Axios 5 秒超时。
