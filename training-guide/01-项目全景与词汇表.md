# 项目全景与词汇表

## 技术栈

| 层次 | 本项目实际使用 | 用途 |
|---|---|---|
| 前端 | Vue 3.5、TypeScript 5.9、Vite 7 | 浏览器页面与交互 |
| 前端组件/样式 | Element Plus、Tailwind CSS 4、自定义 CSS | 表单、弹窗、布局与主题 |
| 前端辅助 | Vue Router、Pinia、Axios、Day.js、Motion-v | 路由、状态、请求、日期、动画 |
| 后端 | Java 17、Spring Boot 3.5.3、Maven | HTTP API 与业务逻辑 |
| 数据层 | Spring Data JPA/Hibernate、MySQL 8 | 数据读写与表映射 |
| 部署 | Ubuntu 22.04、Nginx、systemd、GitHub Actions | 托管、守护与自动发布 |

注意：`pom.xml` 同时显式声明 Spring Security 3.3.4，而 Spring Boot 父版本是 3.5.3，并有重复依赖。后续维护宜移除显式版本和重复项，让 Boot 统一管理兼容版本。

## 一次请求如何流动

以“前端查询课题”为例：页面调用 `src/api/course.ts` → Axios 将地址变成 `/api/course/...` → 开发时 Vite 把请求代理到 `localhost:8080`，生产时 Nginx 代理 → `CourseController` 接收 → `CourseService` 处理 → `CourseRepository` 访问 MySQL → JSON 原路返回页面。

排错时从这条链逐段确认，不要一次修改多层。

## 主要业务模块

- `identity`：账户、密码、角色和逻辑删除。
- `profile`：头像、简介、称号与审核状态。
- `course`：课题主体，类别包括普通、比赛、Bingo、排名等。
- `issue`：课题内部的题目、歌曲、图片与文件。
- `score`：用户提交、审核、得分、备注与排名数据。
- `activity` + `activity_course_assoc`：活动及其关联课题。
- `rank_config`：排名计分配置。
- `team` + `bingo_tips`：Bingo 队伍、格子状态和提示。

后端 API 主要前缀是 `/api/identity`、`/api/profile`、`/api/course`、`/api/issue`、`/api/score`、`/api/activity`、`/api/rank`、`/api/team` 和 `/api/bingo-tip`。

## 新人必须认识的词

- 终端：输入命令的窗口（PowerShell、Terminal）。
- 当前目录：命令正在操作的文件夹；先用 `pwd` 确认。
- 依赖：项目借用的第三方代码；前端由 pnpm 管理，后端由 Maven 管理。
- 构建：把源代码检查并转成可部署产物；前端是 `dist/`，后端是 `target/*.jar`。
- API：前端与后端约定的请求地址、参数和返回格式。
- 端口：同一电脑上区分服务的号码；本项目 8081 是前端、8080 是后端。
- 环境/Profile：开发和生产的不同配置；Maven 的 `dev` 默认激活，打生产包使用 `-P prod`。
- CORS：浏览器对跨来源请求的限制；开发环境由后端允许 8081，生产由 Nginx 同源代理。
- ORM/JPA：用 Java Entity 映射数据库表。
- CI/CD：推送代码后由 GitHub Actions 自动构建和部署。
- 日志：程序运行记录；报错时最重要的证据。
