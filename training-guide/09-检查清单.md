# 检查清单

## 新人首次上手完成标准

- [ ] 会用终端进入正确目录并用 `pwd`/`Get-Location` 确认。
- [ ] 能说明前端、后端、数据库、Nginx 分别做什么。
- [ ] 用本地数据库启动后端，不连接生产库。
- [ ] 启动前端并完成注册/登录/目标页面访问。
- [ ] 会从浏览器 Network 找到失败请求，从后端日志找异常。
- [ ] 会创建分支、查看 diff、小步提交并发起 PR。
- [ ] 知道秘密和生产数据不能提交给 Git 或发送给 AI。

## 每次开发提交前

- [ ] `git status` 中只有本任务文件，没有 `.env`、密码、数据库导出或构建产物。
- [ ] `git diff` 已逐行读过，没有调试日志和无关格式化。
- [ ] 前端改动已运行 `pnpm build`。
- [ ] 后端改动已运行 Maven 测试/打包。
- [ ] 已手测成功、失败、空数据、权限和重复点击。
- [ ] API/数据库变更已同步前后端、测试和文档。
- [ ] PR 写明验证方式、风险和回滚。

## 发布到 main 前

- [ ] 至少一名维护者批准，分支保护与 Actions 环境审批生效。
- [ ] 没有把数据库凭据、私钥、Token、真实导出加入提交或日志。
- [ ] 数据库变更已备份、在副本演练，并有兼容/回滚计划。
- [ ] 权限变更验证未登录 401、普通用户 403、管理员成功。
- [ ] 上传功能验证类型、大小、路径、磁盘和备份。
- [ ] 清楚本次 main push 会自动部署环境、后端和 main 前端。
- [ ] 安排发布后观察者与明确回滚版本。

## 发布后五分钟

- [ ] GitHub Actions 全部成功；失败时没有盲目重复运行。
- [ ] Nginx 配置与服务正常，后端 8080 已就绪。
- [ ] 首页、登录、一个普通业务页和一个管理员页正常。
- [ ] 关键新增功能和相邻旧功能正常。
- [ ] 日志没有新的持续异常，CPU/内存/磁盘无明显异常。
- [ ] 记录发布 commit、时间、验证人和异常。
