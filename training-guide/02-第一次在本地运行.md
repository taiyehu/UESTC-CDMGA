# 第一次在本地运行

## 1. 安装软件

推荐 Windows 11 + VS Code；也可用 Ubuntu 22.04/WSL2。安装 Git、Java 17、Node.js 20、MySQL 8。项目自带 Maven Wrapper，不强制全局安装 Maven。

验证：

```powershell
git --version
java -version
node --version
corepack enable
corepack prepare pnpm@10.30.0 --activate
pnpm --version
mysql --version
```

预期：Java 主版本为 17，Node 主版本为 20，pnpm 为 10.30.0。版本明显不同先不要继续。

## 2. 准备本地数据库

不要让新人连接生产数据库。创建独立本地库和低权限账号：

```sql
CREATE DATABASE cdmga_dev CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_ai_ci;
CREATE USER 'cdmga_dev'@'localhost' IDENTIFIED BY '<本地强密码>';
GRANT ALL PRIVILEGES ON cdmga_dev.* TO 'cdmga_dev'@'localhost';
FLUSH PRIVILEGES;
```

`sql_script` 同时存在历史建表脚本、增量脚本和包含实际数据的完整导出，不能把它们全部依次执行。新建空库时应由维护者提供一份确认过且脱敏的基线脚本；在此之前可让 Hibernate 创建基础结构，但结果未必涵盖所有索引和外键。

## 3. 创建仅供本机使用的后端配置

当前 `application.properties` 被 `.gitignore` 忽略但已存在于工作区。先备份，然后改为本地连接；绝不提交真实密码。推荐把值写成环境变量占位：

```properties
spring.datasource.url=${DB_URL:jdbc:mysql://localhost:3306/cdmga_dev?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Shanghai}
spring.datasource.username=${DB_USER:cdmga_dev}
spring.datasource.password=${DB_PASSWORD:请替换}
spring.jpa.hibernate.ddl-auto=update

course.upload.directory=${UPLOAD_IMAGES:./local-upload/images/}
score.upload.directory=${UPLOAD_SCORES:./local-upload/scores/}
avatar.upload.directory=${UPLOAD_AVATARS:./local-upload/avatars/}
activity.upload.banner-directory=${UPLOAD_BANNERS:./local-upload/banners/}
activity.upload.file-directory=${UPLOAD_FILES:./local-upload/files/}
```

Windows 无法直接写 `/var/webserver/...`，因此本地必须改上传目录并创建这些文件夹。启动目录为 `webpage` 时，相对路径位于 `webpage/local-upload`。

## 4. 启动后端

打开第一个终端：

```powershell
cd D:\projects\webpage\webpage
.\mvnw.cmd spring-boot:run
```

看到类似 `Started WebpageApplication` 后不要关闭终端。访问 `http://localhost:8080/hello`；有响应说明 HTTP 服务已启动。若数据库连接失败，优先检查 MySQL 是否启动、库名/账号/密码和 3306 端口。

## 5. 启动前端

打开第二个终端：

```powershell
cd D:\projects\webpage\vueproject
pnpm install --frozen-lockfile
pnpm dev
```

首次安装如提示批准构建脚本，运行 `pnpm approve-builds`，审阅后批准项目实际依赖，再执行 `pnpm rebuild`。浏览器打开 `http://localhost:8081`。

## 6. 第一次验收

- 后端终端没有持续红色异常。
- 前端终端显示 Local 地址且浏览器能打开登录页。
- 浏览器按 F12 → Network，业务请求发往 `/api/...`，状态不是连接失败。
- 注册一个纯测试账号并能登录；不要使用生产账号或真实个人信息。
- 修改代码前执行一次 `pnpm build` 和后端测试，记录原始失败，避免把旧问题误认为自己造成。

## 常见启动错误

| 现象 | 原因/处理 |
|---|---|
| 8080/8081 已被占用 | 关闭旧进程，或查占用：`Get-NetTCPConnection -LocalPort 8080` |
| `pnpm` 不存在 | 启用 Corepack 并激活 10.30.0 |
| 前端请求 500 | 看后端终端最早的 `Caused by`，通常是数据库/数据问题 |
| 前端请求 404 | 检查 API 地址和 Controller 映射，不要随意删 `/api` |
| CORS 报错 | 确认前端从 8081 访问，后端 dev 配置允许该来源 |
| 上传失败 | 检查本地上传目录存在且可写；生产还要检查 Nginx 限制 |
